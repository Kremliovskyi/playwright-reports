# Playwright Report Viewer

A lightweight, modern Node.js dashboard to view your Playwright test reports (with full trace functionality) without needing a local Playwright installation. It runs reliably in the background using PM2.

This project is fully compatible with **Windows, macOS, and Linux**.

## Prerequisites

- **Node.js**: Ensure Node.js is installed on your Windows machine (download from [nodejs.org](https://nodejs.org/)).
- **NPM**: Comes installed with Node.js.

## Initial Setup

1. Clone or copy this repository to your Windows machine.
2. Open a terminal (PowerShell or Command Prompt) and navigate to the project directory.
3. Install the required dependencies:
   ```cmd
   npm install
   ```
4. Install PM2 globally (recommended for background process management):
   ```cmd
   npm install -g pm2
   ```

## Adding Your Reports

To view actual Playwright reports:

1. Make sure you have a `reports/` folder in the root of this project. If not, create it:
   ```cmd
   mkdir reports
   ```
2. Copy any `playwright-report` folder generated by Playwright into the `reports/` directory. For example:
   ```text
   playwright-reports/
   └── reports/
       ├── my-e2e-run-frontend/
       │   └── index.html
       └── my-k6-api-load-tests/
           └── index.html
   ```

## Starting the Server (Background)

Use PM2 to start the server in the background so you don't have to keep the terminal window open:

```cmd
pm2 start ecosystem.config.js
```

Once started, open your browser and navigate to:
**👉 http://localhost:3000**

## Essential PM2 Commands

Here are the most common PM2 commands you'll need to manage the viewer:

- **Check Status / See if it's running:**

  ```cmd
  pm2 status
  ```

- **View Live Logs:**

  ```cmd
  pm2 logs playwright-reports
  ```

- **Restart the Server:**
  _(Use this if the server ever crashes or you manually changed the `server.js` file)_

  ```cmd
  pm2 restart playwright-reports
  ```

- **Stop the Server:**
  _(Stops the server but keeps it in the PM2 list)_

  ```cmd
  pm2 stop playwright-reports
  ```

- **Remove the Server from PM2 entirely:**

  ```cmd
  pm2 delete playwright-reports
  ```

- **Make PM2 auto-start the server on Windows Reboot:**
  To ensure the dashboard boots up automatically when your Windows machine restarts, you can install the `pm2-windows-startup` package:
  ```cmd
  npm install pm2-windows-startup -g
  pm2-startup install
  pm2 save
  ```

## Troubleshooting

- **No reports appearing?** Make sure the inner folders inside `/reports` actually contain an `index.html` file right at the top level of that folder.
- **Port in use?** If port `3000` is already taken, you can change the port in `ecosystem.config.js`.
